# 레지스터 메모리

레지스터는 컴퓨터에서 가장 작고 가장 빠른 저장 장치다. 메인 메모리(RAM)의 일부가 아니라 CPU 내부에 직접 위치하며, CPU가 지금 당장 처리 중인 데이터·명령어·주소를 담아둔다. 모든 데이터는 연산되기 전에 반드시 레지스터를 거친다 — RAM이 아무리 빨라도 ALU가 직접 계산할 수 있는 대상은 레지스터에 있는 값뿐이기 때문이다.

레지스터 하나의 용량은 보통 32~64비트 수준으로 매우 작다. 대신 CPU 코어 바로 옆에 있어서 접근 지연이 사실상 0에 가깝다. "CPU가 빠르다"는 말은 상당 부분 레지스터가 얼마나 많고 얼마나 넓은가와 관련이 있다 — 레지스터가 부족하면 CPU는 자주 메모리를 오가야 하고, 그때마다 RAM 접근 지연이 그대로 성능 손실이 된다.

레지스터-메모리 아키텍처에서는 연산의 피연산자가 레지스터에만 있을 수도(레지스터 전용), 레지스터와 메모리에 하나씩 걸쳐 있을 수도 있다. 어느 쪽이든 메모리 접근 없이 처리할 수 있는 연산이 많을수록 빠르다.

## 컴퓨터 레지스터의 종류와 역할

용도가 겹치지 않도록 레지스터는 목적별로 나뉜다.

| 레지스터 | 이름 | 역할 |
|---|---|---|
| DR | Data Register | 연산에 사용할 피연산자, 주변장치로 오가는 데이터를 임시 저장 |
| PC | Program Counter | 다음에 fetch할 명령어의 주소 보관, 명령어 실행마다 자동 증가 |
| IR | Instruction Register | 메모리에서 가져온 "현재 실행할" 명령어 자체를 저장 |
| AC | Accumulator | 연산 결과가 쌓이는 레지스터 |
| AR | Address Register | 메모리에서 읽거나 쓸 위치의 주소를 저장 |
| I/O AR | I/O Address Register | 특정 입출력 장치의 주소 지정 |
| I/O BR | I/O Buffer Register | CPU와 I/O 모듈 사이 데이터 교환 버퍼 |

이 표에서 헷갈리기 쉬운 두 쌍이 있다.
- **IR vs PC**: PC는 "다음에 가져올 명령어의 *주소*"를, IR은 "방금 가져온 명령어 *자체*"를 담는다. PC가 가리키는 곳에서 읽어온 값이 IR에 들어간다.
- **AR vs AC**: AR은 메모리 "위치"를, AC는 연산 "결과값"을 담는다. 하나는 주소, 하나는 데이터라는 점이 다르다.

## 교육용 모델: 16비트, 4096워드

대학 컴퓨터구조 수업에서 흔히 쓰이는 Morris Mano의 교육용 모델은 16비트 워드, 2¹² = 4096개의 메모리 주소를 사용한다. 이 모델에서 각 레지스터의 비트 수는 담는 정보의 종류에 따라 다르게 설계된다.

| 레지스터 | 비트 수 | 이유 |
|---|---|---|
| DR, IR, AC | 16비트 | 워드 크기(16비트)와 동일한 데이터를 다루므로 |
| AR, PC | 12비트 | 4096(2¹²)개의 주소만 표현하면 되므로, 데이터보다 작아도 됨 |

AR과 PC가 같은 12비트인 이유도 단순하다. 둘 다 "주소"를 담는 레지스터이고, 이 모델의 주소 공간이 4096칸이라 12비트면 충분하기 때문이다. 반대로 DR·IR·AC는 실제 데이터(워드)를 다루므로 워드 크기와 같은 16비트가 필요하다.

## 핵심 정리
- 레지스터는 "CPU가 지금 만지고 있는 값"이 머무는 자리이고, 종류별로 담는 정보(주소 vs 데이터)가 다르다.
- 레지스터 수가 많고 넓을수록 메모리 접근 빈도가 줄어 CPU가 빨라진다.
- 레지스터의 비트 수는 "무엇을 담는가"(주소냐 데이터냐)에 따라 결정된다 — 주소용 레지스터는 주소 공간 크기만큼만, 데이터용 레지스터는 워드 크기만큼 필요하다.
